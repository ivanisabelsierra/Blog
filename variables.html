Las variables permiten asignar nombres coherentes a información para ser reutilizada con facilidad. En Python, no se establece el tipo de dato a la hora de declarar una variable, realmente no existe una declaración propiamente dicha como existe en otros lenguajes, simplemente escribes el nombre que quieres que tenga y le asignas el valor correspondiente.


Asignaciones

La forma de dar valor a una variable se realiza por asignaciones y, en Python, pueden ser simples o múltiples.

En Python las asignaciones se hacen utilizando el carácter = y la dirección es hacia la izquierda, por tanto, se guarda en la parte izquierda lo que se ponga a la derecha.

En la parte izquierda hay que situar un identificador válido y a la derecha un literal o una expresión, que serán los datos a asignar al identificador.

variable = [literal o expresión]

Cuando se hacen asignaciones, se pueden utilizar dos tipos de datos:

  • Literales: son datos descritos directamente en el código. Es información estática, puede ser simple o usar instancias de clases. La definición clave es que se conoce cuál será el valor asignado analizando el código.

  • Expresiones: son datos asignados tras la evaluación de una o varias sentencias. Suelen ser dinámicos y no se conoce el valor final analizando el código.

Nombres variables

Los nombres de las variables de Python deben de escribirse en snake_case. Snake case es un estilo de escritura en el que cada espacio se reemplaza con un carácter de subrayado: ‘_’, y la primera letra de cada palabra se escribe en minúsculas.

Además, deben de cumplir las siguientes características:

  • Tienen que empezar por una letra o barra baja.
  • El uso de keywords como nombres está prohibido.
  • Los nombres deben de ser descriptivos.
  • Deben de estar en minúsculas y separando palabras por barras bajas ‘_’.
  • Las constantes se escriben en mayúsculas.

Operadores

Los operadores son símbolos que le indican al intérprete que realice una operación específica. Los operadores se agrupan según la función que realizan:

Operadores aritméticos

Los operadores aritméticos son aquellos que nos permiten realizar operaciones aritméticas con los datos. Tenemos los siguientes:

  • ‘+’ realiza la suma entre dos operandos

12 + 3 = 15

  • ‘-‘ realiza la resta entre dos operandos

12 – 3 = 9

  • ‘*’ realiza la multiplicación entre dos productos

12 * 3 = 36

  • ‘/’ realiza la división entre dos operandos

12 / 3 = 4

  • ‘%’ devuelve el resto de la división entre dos operandos

16 % 3 = 1

  • ‘**’ realiza la potencia entre dos operandos

12 ** 3 = 123 = 1728

  • ‘//’ devuelve la parte entera del resultado de la división entre dos operandos

18 // 5 = 3

Operadores relacionales

Los operadores relacionales son aquellos que nos van a permitir realizar comparaciones entre dos elementos y devolverá un valor booleano:

-      True: la comparación se cumple

-      False: la comparación no se cumple

Los operadores relacionales son:

  • ‘>’ menor que

5 < 6 => True

  • ‘>’ mayor que

5 > 6 => False

  • ‘<=’ menor o igual que

5 >= 6 => False 

  • ‘<=’ mayor o igual que 
6 >= 6 => True
  • ‘==’ igual que

5 == 6 => False

  • ‘!=’ distinto que

5 != 6 => True

Operadores lógicos

Los operadores lógicos permiten combinar las operaciones relacionales o valores booleanos independientes para obtener un único resultado; es decir, devuelve un resultado en función de múltiples condiciones. Los operadore lógicos son:

  • ‘and’ realiza la operación lógica ‘Y’ entre dos elementos; es decir, el resultado será True si ambos elementos son True, de lo contrario será False

(5<3) and (4 == 7) => False

  • ‘or’ realiza la operación lógica ‘O’ entre dos elementos; es decir, el resultado será True si alguno de los elementos es True, de lo contrario será False

(5>3) and (4 == 7) => True

  • ‘not’ realiza la operación lógica ‘NO’; es decir, el resultado será True el elemento es False y viceversa

not (6==7) => True

Operadores de asignación

El operador de asignación: ‘=’, sirve para asignar un valor a una variable. Para ello, lo que esté en la parte derecha del operador será almacenado en la variable de la izquierda.

Variable = valorVariable

Algunos ejemplos son:

El valor 923 será asignado a la variable Precio

La cadena de texto “Moreno” se almacenará en la variable apellido

El numero 34 será guardado en la variable numero

No obstante, el operador: ‘=’, no es el único operador de asignación que hay ya que existen otros operadores que derivan de este y tienen un funcionamiento parecido a este:

*Sea a = 2 para todos los casos:

  • ‘+=’ suma el valor de la derecha del operador al valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a += 5 es equivalente a: a = a+5, el valor que almacenará la variable a al final es: 7.

  • ‘-=’ resta el valor de la derecha del operador al valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a -= 5 es equivalente a: a = a-5, el valor que almacenará la variable a al final es: -3.

  • ‘*=’ multiplica el valor de la derecha del operador al valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a *= 5 es equivalente a: a = a*5, el valor que almacenará la variable a al final es: 10.

  • ‘/=’ divide el valor de la derecha del operador al valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a /= 5 es equivalente a: a = a/5, el valor que almacenará la variable a al final es: 0,4.

  • ‘%=’ calcula el resto de la división entre el valor de la izquierda del operador y el valor de la variable a la derecha y luego asigna el resultado a esa misma variable.

La operación: a %= 5 es equivalente a: a = a%5, el valor que almacenará la variable a al final es: 2.

  • ‘**=’ eleva el valor de la variable a la izquierda a la potencia del valor de la derecha y luego asigna el resultado a esa misma variable.

La operación: a **= 5 es equivalente a: a = a**5, el valor que almacenará la variable a al final es: 25 = 32.

  • ‘//=’ realiza una división entera (se queda solo con la parte entera del resultado) entre el valor de la izquierda del operador y el valor de la variable a la derecha y luego asigna el resultado a esa misma variable.

La operación: a //= 5 es equivalente a: a = a//5, el valor que almacenará la variable a al final es: 0.

  • ‘&=’ realiza una operación AND bit a bit entre el valor de la derecha del operador y el valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a &= 5 es equivalente a: a = a&5, el valor que almacenará la variable a al final es: 0.

  • ‘|=’ realiza una operación OR entre el valor de la derecha del operador y el valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a |= 5 es equivalente a: a = a|5, el valor que almacenará la variable a al final es: 7.

  • ‘^=’ realiza una operación XOR el valor de la derecha del operador y el valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a ^= 5 es equivalente a: a = a^5, el valor que almacenará la variable a al final es: 7. 

  • ‘>>=’ realiza un desplazamiento a la derecha de los bits de la variable a la izquierda del operador, el número de bits desplazados será el valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a >>= 5 es equivalente a: a = a>>5, el valor que almacenará la variable a al final es: 0.

  • ‘<<=’ realiza un desplazamiento a la izquierda de los bits de la variable a la izquierda del operador, el número de bits desplazados será el valor de la variable a la izquierda y luego asigna el resultado a esa misma variable.

La operación: a <<= 5 es equivalente a: a = a<<5, el valor que almacenará la variable a al final es: 64.

Operadores de pertenencia

Los operadores de pertenencia se emplean para comprobar si un valor está presente dentro de una colección, ya sea tupla, conjunto, diccionario o incluso una cadena de texto. Los operadores de pertenencia son:

  • ‘in’ devuelve True si el valor especificado se encuentra dentro de la colección, en contrario devolverá False

Siendo: a = [1,2,3,4,5], realizamos: 6 in a, resultado: False

Siendo: b = “Hello World”, realizamos: “World” in b, resultado: True

  • ‘not in’ devuelve False si el valor especificado no se encuentra dentro de la colección, en contrario devolverá False

Siendo: a = [1,2,3,4,5], realizamos: 6 not in a, resultado: True

Siendo: b = “Hello World”, realizamos: “World” not in b, resultado: False

Operadores de pertenencia

Los operadores de pertenencia se emplean para comparar la identidad de los objetos, no sus valores; es decir, verifican si dos variables se refieren al mismo objeto en memoria. Hay que tener cuidado ya que, aunque haya dos variables que posean el mismo valor primitivo o sean colecciones con los mismos elementos, si estas variables no apuntan a la misma dirección de memoria, no serán los mismos objetos. Los operadores de pertenencia son:

  • ‘is’ comprueba si dos variables se refieren al mismo objeto en memoria devolviendo True en este caso, sino devuelve False
  • ‘is not’ comprueba si dos variables no se refieren al mismo objeto en memoria devolviendo True en este caso, sino devuelve False

Algunos ejemplos son:

a)     a = [1,2,3]

b = a

a is b => True

a is not b => False

b)     a = [1,2,3]

b = [1,2,3]

a is b => False 

a is not b => True

c)     a = 1000

b = 1000

a is b => False

a is not b => True


Mutabilidad

Cuando igualamos una variable a otra variable, estamos asignando que la variable a la izquierda del operador ‘=’ apunte a la dirección de memoria de la variable a la derecha del operador. Por lo tanto, como están apuntando a la misma dirección de memoria, poseerán el mismo valor las dos.

No obstante, debemos tener en cuenta que hay dos tipos de variables:

  • Variables inmutables: que una variable sea inmutable significa que el valor del objeto no puede ser cambiado una vez se haya creado. Si quieres modificar el valor de estas variables, se necesita crear un nuevo objeto. Dentro de este tipo de variables encontramos los datos primitivos como datos enteros, las cadenas de caracteres o las tuplas.

Por lo tanto, al asignar una variable inmutable a otra simplemente se copia la referencia al mismo objeto en memoria. Luego, si a una de estas dos variables se le establece un nuevo valor se creará una nuevo objeto con una nueva referencia, sin afectar a la referencia original
